# 代码提交规范

### 提交信息格式

提交信息应遵循以下格式：

```
<类型>: <描述>

[可选的详细描述]
```

#### 类型

- `feat`: 新增功能
- `fix`: 修复bug
- `docs`: 文档相关修改
- `style`: 代码格式调整（不影响代码功能）
- `refactor`: 代码重构（既不是新增功能也不是修复bug）
- `test`: 测试相关修改
- `chore`: 构建过程或辅助工具的变动

#### 描述

- 简要描述本次提交的主要内容，使用祈使句，首字母小写，不使用句号结尾。

### 示例

```
feat: 添加用户登录功能

本次提交新增了用户登录功能，包括登录页面和后端接口。
```

请确保每次提交信息都符合以上规范，以便更好地管理项目。
